Restaurant Food Delivery — Microservices on AWS EKS

Cloud-native microservices with Spring Boot and an Angular frontend. Automated CI/CD (Jenkins → Docker) with GitOps (ArgoCD) to AWS EKS (ALB). Service discovery via Eureka. Data split across MySQL (RDS) and MongoDB Atlas.

Tech

Spring Boot Angular Eureka Docker Jenkins ArgoCD AWS EKS + ALB MySQL (RDS) MongoDB Atlas Postman SonarQube/JaCoCo

Services
  • FoodCatalog — Items, categories, pricing; JPA + MySQL (RDS).
  • User — Registration/login, profile; JPA + MySQL (RDS).
  • Restaurant — Outlet details, menus, availability; JPA + MySQL (RDS).
  • Orders — Cart, order workflow; stored in MongoDB Atlas for flexibility.
Platform & Pipeline
  • Build: Jenkins pipelines compile, test (JUnit/JaCoCo), and run SonarQube quality gates.
  • Containerize: Docker images pushed to registry; versioned by Git SHA.
  • Deploy: ArgoCD syncs K8s manifests to AWS EKS; traffic via ALB Ingress.
  • Operate: Health checks, rolling updates; smoke tests with Postman; logs in Splunk (if configured).
Highlights / Impact
  • Modular microservices with clear bounded contexts and independent deploys.
  • Zero-touch deployments after code merge using GitOps.
  • Data split by access patterns: relational (RDS) vs. document (MongoDB Atlas).
  • Quality gates enforced (SonarQube + JaCoCo) before containerization.
